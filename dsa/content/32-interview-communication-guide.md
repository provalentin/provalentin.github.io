# Interview Communication Guide

**Read time:** 10 min | **Video:** 12 min

## Before Coding

1. **Clarify Requirements**
   - "Is the array sorted?"
   - "Can values be negative?"
   - "What's the input range?"
   - "Do I need to modify in-place?"

2. **Discuss Approach**
   - "First, let me think of a brute force O(n²) solution"
   - "Then I can optimize to O(n) using a hash map"
   - "Should I code the optimized version?"

3. **State Complexity**
   - "This approach is O(n) time, O(1) space"
   - "Alternative is O(log n) with O(n) space"

## While Coding

1. **Narrate Your Thinking**
   - "I'm initializing a counter"
   - "Now I'll iterate through the array"
   - "For each element, I check if..."

2. **Ask for Validation**
   - "Does this approach make sense?"
   - "Should I add error handling here?"

3. **Test as You Go**
   - Walk through example while coding
   - Verify logic before finishing

## After Coding

1. **Test With Examples**
   - "Let me trace through example: [1,2,3]"
   - "Empty array: returns 0"
   - "Single element: returns 1"

2. **Discuss Trade-offs**
   - "This uses O(n) space but runs faster"
   - "Alternative uses O(1) space but slower"

3. **Optimize If Time**
   - "Can we improve the constant factor?"
   - "Any edge cases we missed?"

## Communication Tips

✓ Speak clearly and confidently
✓ Don't rush
✓ Pause to think
✓ Admit when stuck
✓ Ask for hints
✓ Explain why you chose an approach

❌ Don't code without thinking first
❌ Don't ignore interviewer's hints
❌ Don't just say "it works"
❌ Don't get defensive
❌ Don't memorize solutions

---

**Key:** Communication > Code. Show your thinking process.
