# Final Interview Checklist

**Read time:** 7 min | **Printable:** Yes

## Before Interview

- [ ] Got 8+ hours of sleep
- [ ] Ate a good breakfast
- [ ] Tested internet connection
- [ ] Closed unnecessary tabs/apps
- [ ] Have water nearby
- [ ] Quiet space, no interruptions
- [ ] Camera and microphone working
- [ ] Resume/notes ready (if allowed)

## During Interview - Setup (5 min)

- [ ] Camera on, good lighting
- [ ] Clear audio, no background noise
- [ ] Share screen (if coding)
- [ ] Have notepad visible for notes
- [ ] Smile - you're on camera!

## During Interview - Problem (2 min)

- [ ] Listen carefully to problem
- [ ] Take notes on constraints
- [ ] Ask clarifying questions
- [ ] Confirm understanding

## Before Coding (3 min)

- [ ] Discuss brute force approach
- [ ] Explain optimization strategy
- [ ] Get approval to code
- [ ] State time/space complexity

## Coding (15-20 min)

- [ ] Code clearly, with spacing
- [ ] Use meaningful variable names
- [ ] Handle edge cases
- [ ] Test with examples
- [ ] Check for off-by-one errors

## Testing (3 min)

- [ ] Walk through example
- [ ] Test empty input
- [ ] Test single element
- [ ] Verify complexity
- [ ] Ask: "Can we optimize?"

## Finishing (2 min)

- [ ] Discuss trade-offs
- [ ] Answer follow-up questions
- [ ] Thank interviewer
- [ ] Ask about next steps

## What NOT to Do

❌ Start coding immediately
❌ Rush through explanation
❌ Say "it works" without testing
❌ Argue with interviewer
❌ Give up when stuck
❌ Copy/paste code from memory
❌ Ignore hints

## Common Interview Questions

"Why did you choose this approach?"
- Be ready to justify your choice

"Can you optimize further?"
- Always consider space-time trade-offs

"What are the edge cases?"
- Think about null, empty, single element, large numbers

"How would you test this?"
- Explain your testing strategy

---

**Preparation + Communication + Problem-Solving = Success**
